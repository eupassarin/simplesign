Manual Técnico - SimpleSign.HtmlToPdf

Este documento longo visa testar a quebra de páginas e a geração de PDFs com múltiplas páginas.

Capítulo 1: Introdução

O SimpleSign.HtmlToPdf é um módulo de conversão de HTML para PDF totalmente desenvolvido em .NET, sem dependências externas. Ele foi projetado para ser leve, rápido e fácil de integrar em qualquer aplicação .NET.

A arquitetura do módulo segue o padrão de pipeline: primeiro o HTML é parseado em uma árvore DOM, depois as regras CSS são aplicadas, em seguida o layout é calculado e finalmente o PDF é renderizado.

Este manual cobre todos os aspectos técnicos do módulo, desde a instalação até o uso avançado com customizações de estilo e layout.

Capítulo 2: Instalação e Configuração

Para instalar o SimpleSign.HtmlToPdf, adicione a referência ao pacote NuGet em seu projeto:

dotnet add package SimpleSign.HtmlToPdf

Não há configurações adicionais necessárias. O módulo funciona out-of-the-box com as configurações padrão (página A4, margens de 40pt).

Para projetos que necessitam de customizações, todas as opções são configuráveis através do builder pattern fluente que será descrito no próximo capítulo.

Capítulo 3: API Fluente

O ponto de entrada principal é a classe estática HtmlToPdfConverter. Ela oferece três formas de iniciar uma conversão:

  • Html(string) - A partir de uma string HTML
  • Html(byte[]) - A partir de bytes UTF-8
  • FileAsync(string) - A partir de um arquivo HTML

Após a inicialização, o builder permite encadear configurações:

byte[] pdf = HtmlToPdfConverter
    .Html(htmlContent)
    .WithPageSize(PageSize.A4)
    .WithMargins(30, 25, 30, 25)
    .WithStylesheet(css)
    .WithTitle("Meu Documento")
    .WithAuthor("João Silva")
    .Convert();

Capítulo 4: Tamanhos de Página

O módulo suporta quatro tamanhos de página padrão:

Tamanho Dimensões Uso Comum
A4 210 × 297 mm Documentos gerais, relatórios
Letter 8.5 × 11 in Padrão norte-americano
Legal 8.5 × 14 in Documentos jurídicos
A3 297 × 420 mm Posters, diagramas grandes

Capítulo 5: Margens

As margens podem ser configuradas de forma uniforme ou individual para cada lado da página. O valor é especificado em pontos (1pt = 1/72 polegada).

Margem uniforme: WithMargins(40) aplica 40pt em todos os lados.

Margens individuais: WithMargins(top, right, bottom, left) permite configurar cada lado separadamente.

As margens padrão são de 40pt em todos os lados, o que proporciona um bom equilíbrio entre espaço útil e legibilidade.

Capítulo 6: Estilos CSS

O conversor suporta estilos CSS inline (atributo style) e folhas de estilo externas via WithStylesheet(). As propriedades CSS suportadas incluem:

  • font-size, font-weight, font-style
  • color, background-color
  • margin, padding
  • border, border-collapse
  • width, text-align
  • display (block, inline)

Os seletores CSS suportados incluem seletores de tipo (h1, p, table), seletores de classe (.classe) e seletores de ID (#id).

Capítulo 7: Elementos HTML Suportados

O parser HTML suporta os seguintes elementos de bloco:

  • Headings: h1, h2, h3, h4, h5, h6
  • Texto: p, div, blockquote, pre
  • Listas: ul, ol, li
  • Tabelas: table, thead, tbody, tfoot, tr, th, td
  • Semânticos: header, footer, section, article, main
  • Outros: hr

E os seguintes elementos inline:

  • Formatação: strong, em, span
  • Links: a
  • Quebra: br

Capítulo 8: Metadados do PDF

É possível definir metadados no documento PDF gerado:

  • Title - Título do documento (aparece nas propriedades do PDF)
  • Author - Autor do documento
  • Header - Template de cabeçalho com placeholders {page}, {pages}, {title}, {date}
  • Footer - Template de rodapé com os mesmos placeholders

Capítulo 9: Performance

O módulo foi otimizado para performance, utilizando:

  • Span-based parsing para minimizar alocações
  • FlateDecode compression para reduzir o tamanho do arquivo
  • Immutable builder para thread-safety
  • Async-first API para operações de I/O

Em benchmarks internos, o módulo consegue converter um documento HTML de 10KB em menos de 50ms em hardware moderno.

Capítulo 10: CLI

O SimpleSign CLI inclui o comando html-to-pdf para conversão via linha de comando:

simplesign-cli html-to-pdf input.html -o output.pdf --page-size A4 --margin 40

Opções disponíveis:

  • --output, -o: Caminho do arquivo PDF de saída
  • --page-size, -s: Tamanho da página (A4, Letter, Legal, A3)
  • --margin: Margem uniforme em pontos
  • --margin-top, --margin-right, --margin-bottom, --margin-left: Margens individuais
  • --css: Caminho para arquivo CSS adicional
  • --title: Título do documento PDF
  • --author: Autor do documento PDF
